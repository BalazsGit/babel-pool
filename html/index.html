HOW TO JOIN

Pool and Miner configuration



Set Reward Assigment


Before start pool mining you should set the pool recipient address to the official pool address
BURST-HW9R-3VCP-Y46P-DA5W8

This is important regarding the reward distribution between pool miners and also precondition of pool mining


Open Your Burst Wallet
Go to Reward Assigement



Set Recipient to
BURST-HW9R-3VCP-Y46P-DA5W8


Wait 4 blocks



Set Name


It is recommended to set the name of the account to customize it
You can set it by clicking the No Name Set text on the upper left side of the wallet





Plotting


Plotting free storage capacity on your HDD or SSD is essential for mining

Plots contains hashes which are using during deadline calculation

Deadline is a verifiable value which is calculated from your plots by a special algorithm to decide who can mine the next block

If your deadline is the lowest you mine the new block and win the reward

Deadline calculation is similar like winning the raffle where the lowest number win
You have a bounch of raffle amoung others, but the chance to win depends how much raffle you have compare to the whole amount of raffle

The more raffle you have the more chance to win

You can plott your HDD with TurboPlotter_0.10.1v



Mining


Download the latest scavenger 1.7.8 miner from here
https://github.com/PoC-Consortium/scavenger/releases

Unzip it

Open config.yaml file with text editor (e.g. notepad++) and set the proper configuration

url: 'http://openburstpool.ddns.net:8126'

target_deadline: 10000000000

Run miner

CONFIGURE

If you have already joined the pool, you can further configure it by sending text messages from your miner account to the pool account:
{POOL_ACCOUNT}

Share Model

Change your share model to 80 % by sending the following text message (any number from 0 up to 100 is accepted):
share 80

Donation Percent

Change your donation percent to 5 % by sending the following text message (any number from 0 up to 100 is accepted):
donate 5

Minimum Payout

Change your minimum payout to 100 BURST by sending the following text message (any number higher than {MIN_PAYOUT} is accepted):
pay 100

You can also configure multiple parameters with a single message, for instance:
share 80 donate 5 pay 100
